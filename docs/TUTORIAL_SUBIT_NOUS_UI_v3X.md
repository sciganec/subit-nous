# 🧠 SUBIT‑NOUS UI – Туторіал

## Вступ

**SUBIT‑NOUS** – це система, яка перетворює будь-який текст у **256 архетипів** на основі 4 вимірів:

| Вісь | Значення | Біти |
|------|----------|------|
| **WHO** | ME, WE, YOU, THEY | 10, 11, 01, 00 |
| **WHERE** | EAST, SOUTH, WEST, NORTH | 10, 11, 01, 00 |
| **WHEN** | SPRING, SUMMER, AUTUMN, WINTER | 10, 11, 01, 00 |
| **MODE** | STATE, VALUE, FORM, FORCE | 10, 11, 01, 00 |

Кожен текст отримує **8-бітний архетип** (від 0 до 255), а також **soft-профіль** – 8 чисел у діапазоні [-1, 1], які показують ступінь вираженості кожного біта.

Веб-інтерфейс надає доступ до всіх функцій SUBIT‑NOUS у зручному графічному вигляді.

---

## 1. Запуск UI

### Вимоги
- Встановлений Python 3.9+
- Встановлений SUBIT‑NOUS: `pip install subit-nous`
- (Опціонально) Встановлений [Ollama](https://ollama.com) для роботи з агентами

### Запуск

```bash
nous ui --port 8501
```

Або безпосередньо:

```bash
streamlit run src/subit_nous/ui.py --server.port 8501
```

Після запуску відкрийте браузер за адресою: `http://localhost:8501`

---

## 2. Інтерфейс

Після запуску ви побачите головне вікно з **4 вкладками**:

- **🔍 Analyze** – аналіз тексту
- **🤖 Agents** – генерація тексту агентами
- **🔎 Search** – гібридний пошук по проіндексованих документах
- **📊 Profile** – візуалізація soft-профілю

Ліворуч є **панель налаштувань**:
- Вибір моделі Ollama (для агентів)
- Індексація папки для пошуку

---

## 3. Вкладка Analyze – аналіз тексту

### Що робить
Визначає архетип тексту, показує його ID, назву, модус (STATE/VALUE/FORM/FORCE) та soft-профіль.

### Як використовувати

1. Введіть текст у поле (наприклад, *"I think logically about the east in spring"*)
2. Натисніть **🔍 Analyze**
3. Отримаєте:
   - **Archetype ID** – число від 0 до 255
   - **Name** – назва архетипу (наприклад, `MICRO mode` або `LOGOS_ME_EAST_SPRING`)
   - **Mode** – STATE, VALUE, FORM, FORCE (або Mixed)
   - **Soft Profile** – 8-бітний вектор у вигляді стовпчикової діаграми
   - **Radar Chart** – радарна діаграма для візуалізації soft-профілю

### Приклад тексту для тестування

| Текст | Очікуваний модус |
|-------|------------------|
| *"I think logically about the east in spring"* | STATE / MICRO |
| *"We trust our community in the south during summer"* | VALUE / MACRO |
| *"You feel the beauty of autumn in the west"* | FORM / MESO |
| *"They exert power in the north during winter"* | FORCE / META |

---

## 4. Вкладка Agents – генерація тексту

### Що робить
Запускає агента з LLM (локально через Ollama), який переписує або доповнює текст у заданому архетипному стилі.

### Доступні агенти

| Агент | Стиль | Що робить |
|-------|-------|-----------|
| **STATE** | Логічний, фактологічний, структурований | Аналізує об'єктивно, використовує факти та дані |
| **VALUE** | Етичний, довірливий, гармонійний | Оцінює з точки зору моралі та спільноти |
| **FORM** | Естетичний, емоційний, художній | Переписує з увагою до краси та емоцій |
| **FORM** | Естетичний, емоційний, художній | Переписує з увагою до краси та емоцій |
| **FORCE** | Вольовий, силовий, амбітний | Аналізує владні динаміки та стратегії |

### Як використовувати

1. Введіть текст у поле (наприклад, *"Write a short story about AI"*)
2. Виберіть режим: **auto** (автовизначення) або конкретний агент
3. Натисніть **🤖 Generate**
4. Отримайте відповідь від агента

### Приклад

**Вхідний текст:**  
*"The sun is setting over the ocean"*

**Вибір:** `FORM` (художній)

**Відповідь агента:**  
*"Golden light spills across the waves like molten honey, painting the horizon in shades of amber and rose. The day breathes its last warm sigh before night's gentle embrace."*

## 5. Вкладка Search – гібридний пошук

### Що робить
Шукає документи в проіндексованій папці, комбінуючи:
- **Семантичну схожість** (косинусна подібність soft-векторів)
- **Фільтри за архетипом** (MODE, WHO, WHERE, WHEN)

### Перед використанням

1. У **панелі налаштувань (ліворуч)** вкажіть шлях до папки з документами (наприклад, `./demo`)
2. Натисніть **📇 Index Folder** – це проіндексує всі текстові файли (.txt, .md, .py тощо)

### Як використовувати

1. Введіть **пошуковий запит** (наприклад, *"renewable energy"*)
2. Виберіть фільтри (опціонально):
   - **Mode filter** – STATE, VALUE, FORM, FORCE
   - **Who filter** – ME, WE, YOU, THEY
3. Налаштуйте:
   - **Number of results** – скільки результатів показати
   - **Semantic weight (alpha)** – баланс між семантикою (0.5 – збалансовано)
4. Натисніть **🔎 Search**

### Результати

Ви побачите список файлів з:
- **Score** – загальна оцінка релевантності
- **Similarity** – косинусна подібність до запиту
- **Mode / Who** – архетипні характеристики документа

### Приклад пошуку

**Запит:** *"climate change"*  
**Фільтр:** `MODE = STATE`  
**Результат:** технічні звіти про клімат, а не політичні промови чи поезія.

---

## 6. Вкладка Profile – візуалізація soft-профілю

### Що робить
Показує **радарну діаграму** для будь-якого тексту – 8-бітний soft-вектор у візуальному вигляді.

### Як використовувати

1. Введіть текст у поле
2. Натисніть **📊 Generate Radar Chart**
3. Отримайте:
   - **Radar Chart** – кожен біт (b0..b7) як окрема вісь
   - **Raw Values** – числові значення кожного біта

### Інтерпретація

- **b7, b6** – WHO (b7=ME/WE, b6=YOU/THEY)
- **b5, b4** – WHERE (EAST/SOUTH/WEST/NORTH)
- **b3, b2** – WHEN (SPRING/SUMMER/AUTUMN/WINTER)
- **b1, b0** – MODE (STATE/VALUE/FORM/FORCE)

Позитивне значення (+0.5) означає, що біт = 1, негативне (-0.5) – що біт = 0.

## 7. Панель налаштувань (Sidebar)

### Вибір моделі Ollama
- **llama3.2:3b** – легка, швидка, рекомендована
- **llama3.2:8b** – якісніша, потребує більше пам'яті
- **mistral:7b** – хороший баланс
- **gemma3:12b** – потужна, потребує 12+ ГБ RAM

### Індексація папки
- Введіть шлях до папки з документами
- Натисніть **📇 Index Folder**
- Після завершення можна шукати по цій папці

---

## 8. Типові помилки та рішення

| Помилка | Рішення |
|---------|---------|
| **"Port 8501 is not available"** | Використайте інший порт: `--server.port 8502` |
| **"No module named 'streamlit'"** | Встановіть: `pip install streamlit plotly pandas` |
| **Agent не відповідає** | Переконайтеся, що Ollama запущено: `ollama list` |
| **Пошук не знаходить документи** | Спочатку проіндексуйте папку кнопкою **📇 Index Folder** |
| **"Model not found"** | Завантажте модель: `ollama pull llama3.2:3b` |

## 9. Приклади використання

### Сценарій 1: Аналіз тексту
Ви пишете статтю і хочете дізнатися, який у неї архетипний профіль.
→ Вкладка **Analyze** → вставте текст → отримайте soft-профіль та радарну діаграму.

### Сценарій 2: Зміна стилю тексту
У вас є сухий технічний звіт, який потрібно переписати в художньому стилі.
→ Вкладка **Agents** → виберіть `FORM` → вставте текст → отримайте поетичну версію.

### Сценарій 3: Пошук документів за тоном
У папці з 1000 документів потрібно знайти всі, що написані в колективному тоні (WE) та з етичним ухилом (VALUE).
→ Вкладка **Search** → фільтри: `WHO = WE`, `MODE = VALUE` → пошук.

### Сценарій 4: Візуалізація архетипного профілю
Ви досліджуєте, як змінюється архетип тексту залежно від теми.
→ Вкладка **Profile** → порівняйте радарні діаграми для різних текстів.

## 10. Подальші кроки

- **Індексуйте свою папку** з документами – отримайте змогу шукати за архетипами.
- **Експериментуйте з агентами** – спробуйте різні режими на одному тексті.
- **Порівнюйте soft-профілі** – введіть два тексти у вкладку Profile і порівняйте радарні діаграми.
- **Інтегруйте з Ollama** – завантажте більші моделі для кращої якості генерації.
